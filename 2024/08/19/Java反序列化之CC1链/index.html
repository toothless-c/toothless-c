一、环境搭建

JDK版本选择的是jdk8u65 ，创建一个maven项目导入maven依赖

因为直接打开源码，打开的是文件都是是 .class 文件，对于代码的理解不是很友好，难读懂。所以需要把替换掉。openJDK 8u65 下载链接,替换sun包，解压jdk8u65目录下的sec.zip目录.把我们下载的sun目录cv到jdk8u65的src目录下

二、漏洞分析

入口类：该类重写了readObject()方法并且实现了Serializable接口

结尾类：一个能够命令执行的方法

查找利用点：查找哪些地方调用了这个入口类，通过链条最后到达结尾类执行命令

从后往前找，结尾类就是Tranformer接口，这个接口里面有个transform方法。

ctrl + alt + B 查询哪些类实现了Transformer接口

InvokerTransformer类实现了Transformer接口，也继承了继承了继承了Serializable接口。分析一下实现的transform方法

重写的transform()方法接受了一个Object对象，**Class cls = input.getClass();**过调用输入对象 input 的 getClass() 方法来获取 Class 对象,**Method method = cls.getMethod(iMethodName, iParamTypes);使用 Class 对象的 getMethod() 方法来获取一个 Method 对象，来实现获取Class里面method方法。iMethodName 是要调用的方法的名称，iParamTypes 是一个 Class 类型的数组。return method.invoke(input, iArgs);**使用 Method 对象的 invoke() 方法来调用方法。第一个参数 input 是方法调用的接收者，即方法将在这个对象上执行。第二个参数 iArgs 是一个对象数组，包含实际传递给方法的参数值。这的功能点就是任意方法调用

使用InvokerTransformer来弹计算器验证一波

接下来就是寻找谁调用了InvokerTransformer 类中的 transform() 方法，Ctrl+Alt+Shift+F7，选择 All place 查询

其中 TransformedMap 类中存在 checkSetValue() 方法调用了 transform() 方法

接下来分析一下checkSetValue()方法。这个方法接收一个 Object value 作为参数，并调用 valueTransformer 的 transform() 方法来转换这个值。在 TransformedMap 的构造方法中，valueTransformer 是作为参数传入的。但是是因为 TransformedMap 的构造方法作用域是 protected，我们需要去找谁调用了 TransformedMap 的构造方法。在 decorate() 静态方法中创建了 TransformedMap 对象
